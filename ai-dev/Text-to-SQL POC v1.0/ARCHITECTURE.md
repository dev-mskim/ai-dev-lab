## 5. ERP 확장 시 변경 포인트

```
현재 (데모)                       ERP 확장 시
──────────────────────────────────────────────────────
SQLite (파일 DB)           →  Oracle DB
  └ db_setup.py의              └ .env의 DB_TYPE=oracle
    create_engine() 1줄만 변경     연결 정보 입력

샘플 스키마 (4개 테이블)   →  실제 ERP 스키마
  └ schema_loader.py의          └ ERP 테이블/컬럼 정보를
    SCHEMA_INFO 교체               최대한 자세히 작성

Streamlit UI (독립 웹앱)   →  ERP 내장 or REST API 연동
  └ 개발 완료 후 결정
```

## 6. 보안 설계

```
사용자 입력
    ↓
[검증] SELECT 쿼리만 허용
    ↓ 차단: DROP / DELETE / UPDATE / INSERT
    ↓ 허용: SELECT
    ↓
DB 실행
    ↓
결과 반환
```

> ERP 연동 시 추가 예정: 사용자 권한별 접근 테이블 제한
